# 知识库扩充路线

## 内容治理主线

项目内的知识文件分为四层，后续扩充按固定顺序进行：

1. `knowledge_sources/`：保存官方原文、来源登记和中文摘要。
2. `knowledge/evidence_cards/`：把来源约束成可追溯、不可越界的证据主张。
3. `knowledge/decision_cards/`：把证据转换成匹配、回应目标、允许动作和禁止动作。
4. `server/knowledge/`：实际聊天使用的投放层，只接收已经完成来源、证据、决策和评测的内容。

`server/knowledge/` 不是独立事实来源。它与决策卡字段不同，目前仍需人工投放；在自动生成流程完成前，每次投放都必须同时增加测试。

## 当前基线

- 证据卡：33 张，全部为 `machine_draft`。
- 决策卡：40 张，其中问题类型 13、机制 8、微干预 8、回应风格 4、安全规则 7。
- 运行时知识：问题类型 13、机制 16、微干预 10、安全条目 6，共 45 条待专业审校内容。
- 运行时来源覆盖：2 条有明确来源引用，43 条仍需补齐。
- 检索评测：45 条，当前全部通过；这只说明规则符合既定样例，不证明心理学或临床正确性。

## 扩充优先级

### P0：先补质量基础

- 为现有 43 条运行时知识补齐来源引用和对应证据卡。
- 对同名但不同 ID 的运行时条目与决策卡建立明确映射。
- 优先复核安全规则、求助边界、低落、焦虑、睡眠和微干预。
- 为每个主题增加正例、近义表达、混合场景和误报反例。

### P1：低风险常见困扰

- 工作压力与工作环境：第一批已加入，仍待专业复核。
- 丧失与哀伤：第一批已加入，仍待专业复核。
- 社交焦虑与回避：来源已登记，等待 NICE 原文手动补录。
- 惊恐体验、情绪过载、家庭压力、关系结束和重大生活变化。

### P2：只识别并建议求助

- 创伤反应、强迫相关困扰、进食问题、物质使用。
- 躁狂或精神病性体验、家庭暴力和现实人身安全问题。

这些主题在没有专项安全规则和专业复核前，不接入普通微干预。

## 质量门槛

- 普通常见困扰至少需要 1 个官方指南或公共卫生来源，并由另一个高质量来源交叉核对后再考虑投放。
- 安全、高风险或可能改变求助路径的主张至少需要 2 个相互独立的权威来源，并完成相关专业人员复核。
- 公众科普资料可以支持表达范围和求助边界，不能单独支持诊断、治疗选择或效果承诺。
- 所有新增内容默认 `machine_draft` 或 `pending_clinical_review`；只有真实专业复核后才能改变状态。
- 数量增长不作为完成标准。来源覆盖、误报测试、安全边界和审校状态必须一起改善。

## 每批完成条件

1. 官方原文已本地保存，登记信息和访问日期完整。
2. 中文摘要写明可用内容和不可用边界。
3. 证据卡通过 `npm run knowledge:evidence:check`。
4. 决策卡通过 `npm run knowledge:decision:check`。
5. 新增正例、反例和安全混合场景，`npm run eval:knowledge` 全部通过。
6. 投放到运行时后，`npm run knowledge:audit` 和 `npm test` 通过。
7. 专业复核未完成时，文档和产品不得宣称内容已经过临床验证。
