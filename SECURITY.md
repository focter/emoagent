# 安全与隐私说明

本仓库是心理健康觉察对话 Demo，不应直接部署为医疗、治疗或紧急危机服务。安全问题既包括传统软件漏洞，也包括可能导致用户隐私泄露、错误危机分流或绕过回应边界的问题。

## 报告安全问题

如果 GitHub 仓库已启用 Private vulnerability reporting，请通过仓库的 **Security → Report a vulnerability** 私下报告。不要在公开 Issue 中粘贴 API key、访问令牌、真实对话、姓名、联系方式或可复现的个人身份信息。

报告中可以包含受影响版本、最小复现步骤、影响和建议修复，但请使用虚构且去标识化的数据。

## 密钥管理

- 真实密钥只应保存在部署平台的服务端 secret manager 或本地 `.env`。
- 不要使用 `VITE_*` 保存秘密；它们会进入浏览器构建产物。
- 提交前运行 `npm run check:secrets`。
- 一旦密钥可能暴露，应先在供应商端撤销并轮换，再清理文件和 Git 历史。
- `.env.example` 只能包含空值、公开地址和非敏感默认值。

## 对话数据

- 前端会话默认保存在当前标签页的 `sessionStorage`。
- 消息正文会发送给本项目后端；配置真实模型后还会发送给相应模型供应商。
- `ENABLE_SAFE_LOG=true` 只启用字段白名单式元数据日志，不应记录对话正文或身份信息。
- 公开部署者仍必须自行制定隐私告知、数据用途、保留期限、访问控制、删除/导出和未成年人策略。

## 已知安全边界

- 高风险识别主要依赖规则和关键词，可能漏报、误报或被规避。
- 内存限流只适用于单进程 Demo，不能替代网关、WAF、身份认证或分布式配额。
- 当前没有真人危机升级、地区化资源维护或 24 小时响应能力。
- 自动测试不能证明临床有效、安全合规或适用于具体个人。

更完整的缺口与上线判断见 [`PROJECT_STATUS.md`](PROJECT_STATUS.md)。
